
## 1. O que é?

`List<T>` é uma coleção **dinâmica** de elementos do mesmo tipo. Diferente de um array, seu tamanho 5pode aumentar ou diminuir durante a execução da aplicação.

Ela faz parte do namespace `System.Collections.Generic` e é uma das coleções mais utilizadas no C#.

---

## 2. Qual problema resolve?

Resolve a principal limitação dos arrays: o **tamanho fixo**.

Com uma `List<T>`, é possível adicionar, remover e inserir elementos sem precisar criar uma nova coleção, tornando o código mais flexível e fácil de manter. 

---

## 3. Como funciona?

Uma `List<T>` armazena elementos do mesmo tipo e gerencia seu tamanho automaticamente. Sempre que necessário, ela aumenta sua capacidade interna para acomodar novos elementos.

Os elementos continuam sendo acessados por índice, da mesma forma que em um array.

Por exemplo:

```csharp
List<string> alunos = new()
{
    "João",
    "Maria",
    "Pedro"
};

Console.WriteLine(alunos[1]); // Maria

alunos.Add("Carlos");

Console.WriteLine(alunos[3]); // Carlos
```

Neste exemplo:

- `Add()` adiciona um novo elemento ao final da lista.
- Os elementos continuam sendo acessados por índice.
- Não é necessário informar previamente o tamanho da coleção.

---

## 4. Quando usar?

- Quando a quantidade de elementos pode variar.
- Quando será necessário adicionar ou remover itens.
- Quando é necessário acessar elementos por índice.
- Quando deseja uma coleção flexível para a maioria dos cenários.

Na prática, `List<T>` costuma ser a primeira escolha quando não há um motivo específico para utilizar outra coleção.

---

## 5. Exemplo prático

```csharp
List<string> frutas = new()
{
    "Maçã",
    "Banana",
    "Laranja"
};

frutas.Add("Uva");
frutas.Remove("Banana");

foreach (var fruta in frutas)
{
    Console.WriteLine(fruta);
}
```

Saída:

```text
Maçã
Laranja
Uva
```

---

## 6. Principais métodos

|Método|Descrição|
|---|---|
|`Add()`|Adiciona um elemento ao final da lista.|
|`AddRange()`|Adiciona vários elementos de uma vez.|
|`Insert()`|Insere um elemento em uma posição específica.|
|`Remove()`|Remove a primeira ocorrência de um elemento.|
|`RemoveAt()`|Remove o elemento de um índice específico.|
|`RemoveAll()`|Remove todos os elementos que atendem a uma condição.|
|`Clear()`|Remove todos os elementos da lista.|
|`Contains()`|Verifica se um elemento existe na lista.|
|`IndexOf()`|Retorna o índice da primeira ocorrência de um elemento.|
|`Find()`|Retorna o primeiro elemento que atende a uma condição.|
|`FindAll()`|Retorna todos os elementos que atendem a uma condição.|
|`Sort()`|Ordena os elementos da lista.|

---

## 7. Limitações

Apesar de ser muito versátil, `List<T>` não é a melhor escolha para todos os cenários.

- Permite elementos duplicados.
- A busca por um elemento pode ser lenta em listas muito grandes.
- Não realiza buscas por chave como um `Dictionary<TKey, TValue>`.
- Não impede valores repetidos como um `HashSet<T>`.

---

## 8. Resumo

- `List<T>` é uma coleção dinâmica de elementos do mesmo tipo.
- Seu tamanho cresce ou diminui automaticamente.
- Permite adicionar, remover e inserir elementos facilmente.
- Os elementos são acessados por índice, assim como em um array.
- É a coleção mais utilizada quando a quantidade de elementos pode variar durante a execução da aplicação.
