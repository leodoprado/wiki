`Queue<T>` é uma coleção genérica que armazena elementos seguindo o princípio **FIFO (First In, First Out)**, ou seja, **o primeiro elemento que entra é o primeiro a sair**.

Assim como a `List<T>`, seu tamanho é dinâmico, permitindo adicionar e remover elementos durante a execução da aplicação.

---
### 2. Qual problema resolve?

Resolve situações em que os elementos precisam ser processados **na mesma ordem em que foram adicionados**.

É muito utilizada para representar filas de atendimento, filas de impressão, processamento de tarefas e qualquer cenário onde a ordem de chegada deve ser respeitada.

---
### 3. Como funciona?

Uma `Queue<T>` funciona como uma fila de pessoas em um banco ou supermercado.

Novos elementos são adicionados **no final da fila** utilizando o método `Enqueue()`, enquanto as remoções sempre acontecem **no início da fila** através do método `Dequeue()`.

Também é possível visualizar o próximo elemento que será removido utilizando o método `Peek()` sem removê-lo da coleção.

Por exemplo:

```
Queue<string> fila = new();

fila.Enqueue("João");
fila.Enqueue("Maria");
fila.Enqueue("Pedro");

Console.WriteLine(fila.Peek()); // João

Console.WriteLine(fila.Dequeue()); // João

Console.WriteLine(fila.Peek()); // Maria
```

Neste exemplo:

- `Enqueue()` adiciona um elemento ao final da fila.
- `Dequeue()` remove e retorna o primeiro elemento.
- `Peek()` retorna o primeiro elemento sem removê-lo.

---
### 4. Quando usar?

- Quando a ordem de processamento deve seguir a ordem de chegada.
- Quando o primeiro elemento inserido deve ser o primeiro a ser removido.
- Para representar filas de atendimento.
- Para processamento de tarefas em sequência.

Na prática, `Queue<T>` é muito utilizada em sistemas de impressão, processamento de mensagens, filas de atendimento e gerenciamento de tarefas.

---
### 5. Exemplo prático

```
Queue<string> atendimento = new();

atendimento.Enqueue("João");
atendimento.Enqueue("Maria");
atendimento.Enqueue("Pedro");

while (atendimento.Count > 0)
{
    Console.WriteLine($"Atendendo: {atendimento.Dequeue()}");
}
```

Saída:

```
Atendendo: João
Atendendo: Maria
Atendendo: Pedro
```

Observe que os clientes são atendidos exatamente na ordem em que chegaram.

---
### 6. Principais métodos

|Método|Descrição|
|---|---|
|`Enqueue()`|Adiciona um elemento ao final da fila.|
|`Dequeue()`|Remove e retorna o primeiro elemento da fila.|
|`Peek()`|Retorna o primeiro elemento sem removê-lo.|
|`Contains()`|Verifica se um elemento existe na fila.|
|`Clear()`|Remove todos os elementos da fila.|
|`Count`|Retorna a quantidade de elementos armazenados.|

---
### 7. Limitações

Apesar de ser ideal para processamento em ordem de chegada, `Queue<T>` possui algumas limitações.

- Não permite acesso por índice.
- Não é possível remover um elemento específico.
- Apenas o primeiro elemento pode ser removido.
- Não é indicada quando é necessário acessar elementos em posições arbitrárias.

---
### 8. Resumo

- `Queue<T>` é uma coleção baseada no princípio **FIFO (First In, First Out)**.
- O primeiro elemento adicionado é o primeiro a ser removido.
- Os elementos são adicionados com `Enqueue()` e removidos com `Dequeue()`.
- Seu tamanho é dinâmico, permitindo adicionar e remover elementos facilmente.
- É a coleção ideal para representar filas e processar elementos na ordem em que foram inseridos.
