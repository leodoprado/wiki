`LinkedList<T>` é uma coleção genérica composta por **nós (nodes)** conectados entre si. Diferente de uma `List<T>`, seus elementos **não são armazenados em posições sequenciais da memória**, mas sim ligados por referências para o próximo e o elemento anterior.

Assim como a `List<T>`, seu tamanho é dinâmico, permitindo adicionar e remover elementos durante a execução da aplicação.

---
### 2. Qual problema resolve?

Resolve situações em que é necessário realizar **muitas inserções e remoções** de elementos, especialmente no início, no fim ou entre outros elementos da coleção.

Enquanto uma `List<T>` pode precisar deslocar vários elementos ao inserir ou remover um item, uma `LinkedList<T>` apenas ajusta as referências entre os nós, tornando essas operações mais eficientes.

---
### 3. Como funciona?

Uma `LinkedList<T>` é formada por uma sequência de **nós**.

Cada nó armazena:

- Um valor.
- Uma referência para o próximo nó.
- Uma referência para o nó anterior.

Dessa forma, os elementos ficam ligados entre si, formando uma cadeia.

Por exemplo:

```
LinkedList<string> cidades = new();

cidades.AddLast("São Paulo");
cidades.AddLast("Curitiba");
cidades.AddLast("Porto Alegre");

cidades.AddFirst("Florianópolis");

foreach (var cidade in cidades)
{
    Console.WriteLine(cidade);
}
```

Saída:

```
Florianópolis
São Paulo
Curitiba
Porto Alegre
```

Neste exemplo:

- `AddFirst()` adiciona um elemento no início da lista.
- `AddLast()` adiciona um elemento no final da lista.
- Os elementos são percorridos na ordem em que estão ligados.

---
### 4. Quando usar?

- Quando serão realizadas muitas inserções e remoções.
- Quando é necessário inserir elementos no início ou no meio da coleção com frequência.
- Quando a ordem dos elementos é importante.
- Quando não é necessário acessar elementos por índice.

Na prática, `LinkedList<T>` é utilizada em implementações de cache, editores de texto, algoritmos de navegação e estruturas que exigem constantes alterações na ordem dos elementos.

---
### 5. Exemplo prático

```
LinkedList<string> playlist = new();

playlist.AddLast("Música A");
playlist.AddLast("Música B");
playlist.AddLast("Música C");

playlist.Remove("Música B");

foreach (var musica in playlist)
{
    Console.WriteLine(musica);
}
```

Saída:

```
Música A
Música C
```

Observe que o elemento foi removido sem a necessidade de deslocar os demais elementos da coleção.

---
### 6. Principais métodos

| Método          | Descrição                                               |
| --------------- | ------------------------------------------------------- |
| `AddFirst()`    | Adiciona um elemento no início da lista.                |
| `AddLast()`     | Adiciona um elemento no final da lista.                 |
| `AddBefore()`   | Adiciona um elemento antes de um nó específico.         |
| `AddAfter()`    | Adiciona um elemento após um nó específico.             |
| `Remove()`      | Remove um elemento da lista.                            |
| `RemoveFirst()` | Remove o primeiro elemento.                             |
| `RemoveLast()`  | Remove o último elemento.                               |
| `Find()`        | Localiza o primeiro nó que contém um determinado valor. |
| `Contains()`    | Verifica se um elemento existe na lista.                |
| `Clear()`       | Remove todos os elementos da lista.                     |
| `First`         | Retorna o primeiro nó da lista.                         |
| `Last`          | Retorna o último nó da lista.                           |
| `Count`         | Retorna a quantidade de elementos armazenados.          |
|                 |                                                         |

---
### 7. Limitações

Apesar de ser eficiente para inserções e remoções, `LinkedList<T>` possui algumas limitações.

- Não permite acesso por índice (`lista[0]`).
- Para acessar um elemento específico é necessário percorrer a lista.
- Consome mais memória do que uma `List<T>`, pois cada nó armazena referências para os nós vizinhos.
- Não é indicada quando o acesso rápido por posição é um requisito.

---
### 8. Resumo

- `LinkedList<T>` é uma coleção formada por nós ligados entre si.
- Seu tamanho é dinâmico, permitindo adicionar e remover elementos facilmente.
- É muito eficiente para inserções e remoções.
- Não possui acesso por índice, sendo necessário percorrer a lista para localizar elementos.
- É a coleção ideal quando há muitas alterações na estrutura da coleção e poucas necessidades de acesso por posição.
