`HashSet<T>` é uma coleção genérica que armazena elementos **únicos** do mesmo tipo. Diferente de uma `List<T>`, ela **não permite elementos duplicados** e foi projetada para realizar operações de busca, inserção e remoção de forma muito eficiente.

Assim como a `List<T>`, seu tamanho é dinâmico, permitindo adicionar e remover elementos durante a execução da aplicação.

---
### 2. Qual problema resolve?

Resolve a necessidade de armazenar uma coleção onde **não podem existir elementos repetidos**.

Além disso, oferece uma forma rápida de verificar se um elemento já existe na coleção, sendo muito utilizada para eliminar duplicidades e realizar comparações entre conjuntos de dados.

---
### 3. Como funciona?

Um `HashSet<T>` utiliza internamente uma estrutura baseada em **tabelas hash (hash table)** para armazenar seus elementos.

Sempre que um novo elemento é adicionado, o `HashSet<T>` verifica automaticamente se ele já existe na coleção.

Se o elemento já estiver presente, ele **não será adicionado novamente**.

Por exemplo:

```
HashSet<string> frutas = new()
{
    "Maçã",
    "Banana",
    "Laranja"
};

frutas.Add("Maçã");
frutas.Add("Uva");

foreach (var fruta in frutas)
{
    Console.WriteLine(fruta);
}
```

Saída:

```
Maçã
Banana
Laranja
Uva
```

Neste exemplo:

- `"Maçã"` já existia na coleção.
- A segunda chamada para `Add("Maçã")` foi ignorada.
- Apenas `"Uva"` foi adicionada.

---
### 4. Quando usar?

- Quando não podem existir elementos duplicados.
- Quando é necessário verificar rapidamente se um elemento já existe.
- Quando deseja eliminar valores repetidos de uma coleção.
- Quando trabalha com operações entre conjuntos, como união, interseção e diferença.

Na prática, `HashSet<T>` é muito utilizado para armazenar identificadores únicos, permissões, tags, categorias e qualquer situação em que valores repetidos não fazem sentido.

---
### 5. Exemplo prático

```
HashSet<int> numeros = new()
{
    10,
    20,
    30
};

numeros.Add(20);
numeros.Add(40);

foreach (var numero in numeros)
{
    Console.WriteLine(numero);
}
```

Saída:

```
10
20
30
40
```

Observe que o número `20` foi adicionado apenas uma vez.

---
### 6. Principais métodos

|Método|Descrição|
|---|---|
|`Add()`|Adiciona um elemento caso ele ainda não exista.|
|`Remove()`|Remove um elemento da coleção.|
|`Clear()`|Remove todos os elementos.|
|`Contains()`|Verifica se um elemento existe na coleção.|
|`UnionWith()`|Une dois conjuntos, eliminando duplicados.|
|`IntersectWith()`|Mantém apenas os elementos comuns entre dois conjuntos.|
|`ExceptWith()`|Remove os elementos que existem em outro conjunto.|
|`SymmetricExceptWith()`|Mantém apenas os elementos que pertencem a um dos conjuntos, mas não aos dois.|
|`Count`|Retorna a quantidade de elementos armazenados.|

---
### 7. Limitações

Apesar de ser excelente para armazenar elementos únicos, `HashSet<T>` possui algumas limitações.

- Não permite elementos duplicados.
- Não possui acesso por índice.
- Não garante a ordem dos elementos durante a iteração.
- Não é indicado quando é necessário acessar elementos por posição, como ocorre em uma `List<T>`.

---
### 8. Resumo

- `HashSet<T>` é uma coleção de elementos únicos.
- Não permite valores duplicados.
- Seu tamanho é dinâmico, permitindo adicionar e remover elementos facilmente.
- Possui operações muito eficientes para verificar a existência de elementos.
- É a coleção ideal quando o objetivo é garantir que cada elemento exista apenas uma vez ou realizar operações entre conjuntos.
