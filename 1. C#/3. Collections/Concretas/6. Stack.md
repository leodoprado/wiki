`Stack<T>` é uma coleção genérica que armazena elementos seguindo o princípio **LIFO (Last In, First Out)**, ou seja, **o último elemento que entra é o primeiro a sair**.

Assim como a `List<T>`, seu tamanho é dinâmico, permitindo adicionar e remover elementos durante a execução da aplicação.

---
### 2. Qual problema resolve?

Resolve situações em que os elementos precisam ser processados na **ordem inversa em que foram adicionados**.

É muito utilizada para implementar funcionalidades como **desfazer ações (Undo)**, histórico de navegação, chamadas de métodos (Call Stack) e avaliação de expressões matemáticas.

---
### 3. Como funciona?

Uma `Stack<T>` funciona como uma pilha de pratos.

Sempre que um novo prato é colocado na pilha, ele fica no topo. Da mesma forma, ao retirar um prato, o primeiro removido será sempre o que está no topo.

Os elementos são adicionados utilizando o método `Push()` e removidos através do método `Pop()`.

Também é possível visualizar o elemento que está no topo utilizando o método `Peek()`, sem removê-lo da pilha.

Por exemplo:

```
Stack<string> pilha = new();

pilha.Push("João");
pilha.Push("Maria");
pilha.Push("Pedro");

Console.WriteLine(pilha.Peek()); // Pedro

Console.WriteLine(pilha.Pop()); // Pedro

Console.WriteLine(pilha.Peek()); // Maria
```

Neste exemplo:

- `Push()` adiciona um elemento ao topo da pilha.
- `Pop()` remove e retorna o elemento do topo.
- `Peek()` retorna o elemento do topo sem removê-lo.

---
### 4. Quando usar?

- Quando o último elemento inserido deve ser o primeiro a ser removido.
- Para implementar funcionalidades de **Desfazer (Undo)**.
- Para armazenar histórico de navegação.
- Para representar chamadas de métodos e recursão.
- Para processamento de expressões matemáticas.

Na prática, `Stack<T>` é muito utilizada em editores de texto, navegadores, compiladores e em diversos algoritmos que necessitam retornar ao estado anterior.

---
### 5. Exemplo prático

```
Stack<string> historico = new();

historico.Push("Página Inicial");
historico.Push("Produtos");
historico.Push("Carrinho");

Console.WriteLine($"Voltando para: {historico.Pop()}");
Console.WriteLine($"Página atual: {historico.Peek()}");
```

Saída:

```
Voltando para: Carrinho
Página atual: Produtos
```

Observe que a última página visitada é a primeira a ser removida do histórico.

---
### 6. Principais métodos

|Método|Descrição|
|---|---|
|`Push()`|Adiciona um elemento ao topo da pilha.|
|`Pop()`|Remove e retorna o elemento do topo.|
|`Peek()`|Retorna o elemento do topo sem removê-lo.|
|`Contains()`|Verifica se um elemento existe na pilha.|
|`Clear()`|Remove todos os elementos da pilha.|
|`Count`|Retorna a quantidade de elementos armazenados.|

---
### 7. Limitações

Apesar de ser ideal para cenários que seguem o princípio **LIFO**, `Stack<T>` possui algumas limitações.

- Não permite acesso por índice.
- Não é possível remover um elemento específico.
- Apenas o elemento do topo pode ser removido.
- Não é indicada quando é necessário acessar elementos em posições arbitrárias.

---
### 8. Resumo

- `Stack<T>` é uma coleção baseada no princípio **LIFO (Last In, First Out)**.
- O último elemento adicionado é o primeiro a ser removido.
- Os elementos são adicionados com `Push()` e removidos com `Pop()`.
- Seu tamanho é dinâmico, permitindo adicionar e remover elementos facilmente.
- É a coleção ideal para representar pilhas, implementar funcionalidades de **Desfazer (Undo)**, armazenar históricos e controlar chamadas de métodos.
