`Dictionary<TKey, TValue>` é uma coleção genérica que armazena elementos no formato **chave-valor** (_key-value_). Cada elemento é identificado por uma **chave única**, permitindo localizar valores de forma rápida e eficiente.

Assim como a `List<T>`, seu tamanho é dinâmico, permitindo adicionar e remover elementos durante a execução da aplicação.

---
## 2. Qual problema resolve?

Resolve a necessidade de localizar elementos de forma rápida utilizando uma **chave**.

Enquanto uma `List<T>` normalmente exige percorrer seus elementos até encontrar o desejado, um `Dictionary<TKey, TValue>` permite acessar um valor diretamente pela chave, tornando a busca muito mais eficiente.

---
### 3. Como funciona?

Um `Dictionary<TKey, TValue>` armazena cada elemento como um par composto por uma **chave (`TKey`)** e um **valor (`TValue`)**.

Cada chave deve ser **única** dentro da coleção. Caso seja utilizada uma chave já existente, uma exceção será lançada ao tentar adicioná-la.

Os valores são acessados diretamente pela chave, sem a necessidade de conhecer sua posição na coleção.

Por exemplo:

```
Dictionary<int, string> alunos = new()
{
    { 1, "João" },
    { 2, "Maria" },
    { 3, "Pedro" }
};

Console.WriteLine(alunos[2]); // Maria

alunos.Add(4, "Carlos");

Console.WriteLine(alunos[4]); // Carlos
```

Neste exemplo:

- Cada aluno possui uma chave única (`int`).
- Os valores são acessados utilizando a chave.
- Novos elementos podem ser adicionados utilizando o método `Add()`.

---
## 4. Quando usar?

- Quando é necessário localizar elementos por uma chave.
- Quando cada elemento possui um identificador único.
- Quando a velocidade de busca é importante.
- Quando deseja armazenar pares de chave e valor.

Na prática, `Dictionary<TKey, TValue>` é muito utilizado para representar dados como usuários por ID, produtos por código, configurações por nome e qualquer situação em que exista uma chave única para identificar um valor.

---
## 5. Exemplo prático

```
Dictionary<string, string> capitais = new()
{
    { "Brasil", "Brasília" },
    { "Argentina", "Buenos Aires" },
    { "Chile", "Santiago" }
};

capitais.Add("Uruguai", "Montevidéu");

Console.WriteLine(capitais["Brasil"]);
```

Saída:

```
Brasília
```

---
## 6. Principais métodos

|Método|Descrição|
|---|---|
|`Add()`|Adiciona um novo par chave-valor.|
|`Remove()`|Remove um elemento pela chave.|
|`Clear()`|Remove todos os elementos do dicionário.|
|`ContainsKey()`|Verifica se uma chave existe.|
|`ContainsValue()`|Verifica se um valor existe.|
|`TryGetValue()`|Obtém um valor sem lançar exceção caso a chave não exista.|
|`TryAdd()`|Adiciona um elemento apenas se a chave ainda não existir.|
|`Keys`|Retorna uma coleção contendo todas as chaves.|
|`Values`|Retorna uma coleção contendo todos os valores.|
|`Count`|Retorna a quantidade de elementos armazenados.|

---
## 7. Limitações

Apesar de ser extremamente eficiente para buscas por chave, `Dictionary<TKey, TValue>` possui algumas limitações.

- Não permite chaves duplicadas.
- O acesso é realizado pela chave, não pelo índice.
- Não mantém, por contrato, uma ordem específica dos elementos (embora implementações atuais preservem a ordem de inserção, isso não deve ser considerado uma garantia do comportamento da coleção).
- Não é indicado quando há necessidade de armazenar múltiplos valores para a mesma chave.

---
## 8. Resumo

- `Dictionary<TKey, TValue>` armazena elementos no formato **chave-valor**.
- Cada chave deve ser única dentro da coleção.
- Permite localizar elementos de forma rápida utilizando uma chave.
- Seu tamanho é dinâmico, permitindo adicionar e remover elementos facilmente.
- É a coleção ideal quando cada elemento possui um identificador único e a velocidade de busca é importante.
