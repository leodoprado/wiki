`IList<T>` é uma **interface** que representa uma coleção ordenada cujos elementos podem ser acessados por **índice**. Ela estende `ICollection<T>`, adicionando funcionalidades como acesso por posição, inserção e remoção em índices específicos.

A implementação mais comum dessa interface é a `List<T>`, mas outras coleções também podem implementá-la.

---
### 2. Qual problema resolve?

Resolve a necessidade de trabalhar com diferentes tipos de listas que compartilham características como acesso por índice, inserção e remoção de elementos, sem depender de uma implementação específica.

Ao utilizar `IList<T>`, um método pode manipular qualquer coleção indexada que implemente essa interface.

---
### 3. Como funciona?

`IList<T>` define que uma coleção deve permitir o acesso aos seus elementos através de um índice numérico.

Além das operações herdadas de `ICollection<T>`, ela adiciona recursos para inserir, remover e localizar elementos utilizando índices.

Por exemplo:

```
IList<string> alunos = new List<string>
{
    "João",
    "Maria",
    "Pedro"
};

Console.WriteLine(alunos[1]); // Maria

alunos[1] = "Carlos";

Console.WriteLine(alunos[1]); // Carlos
```

Neste exemplo:

- A variável foi declarada como `IList<string>`.
- A implementação utilizada é uma `List<string>`.
- Os elementos são acessados e alterados por índice.
- É possível utilizar os métodos definidos pela interface.

---
### 4. Quando usar?

- Quando um método precisa acessar elementos por índice.
- Quando será necessário inserir ou remover elementos em posições específicas.
- Quando deseja trabalhar com qualquer coleção indexada.
- Quando não deseja depender diretamente da implementação `List<T>`.

Na prática, `IList<T>` é utilizada quando o acesso por índice faz parte dos requisitos do método, mas a implementação concreta da coleção não é importante.

---
### 5. Exemplo prático

```
void ExibirPrimeiroAluno(IList<string> alunos)
{
    if (alunos.Count > 0)
    {
        Console.WriteLine(alunos[0]);
    }
}

IList<string> lista = new List<string>
{
    "João",
    "Maria",
    "Pedro"
};

ExibirPrimeiroAluno(lista);
```

Saída:

```
João
```

Observe que o método recebe um `IList<string>`, sem depender diretamente da classe `List<T>`

---
### 6. Principais membros

|Membro|Descrição|
|---|---|
|`this[int index]`|Obtém ou altera o elemento localizado em um índice específico.|
|`Insert()`|Insere um elemento em uma posição específica.|
|`RemoveAt()`|Remove o elemento de um índice específico.|
|`IndexOf()`|Retorna o índice da primeira ocorrência de um elemento.|
|`Add()`|Adiciona um elemento ao final da coleção.|
|`Remove()`|Remove um elemento da coleção.|
|`Count`|Retorna a quantidade de elementos armazenados.|

---
### 7. Limitações

Apesar de oferecer acesso por índice, `IList<T>` possui algumas limitações.

- Não define como os elementos são armazenados internamente.
- Inserções e remoções podem ter custos diferentes dependendo da implementação utilizada.
- Não oferece funcionalidades específicas de determinadas coleções, como `Enqueue()` (`Queue<T>`), `Push()` (`Stack<T>`) ou `AddFirst()` (`LinkedList<T>`).
- Apenas coleções indexadas podem implementar essa interface.

---
### 8. Resumo

- `IList<T>` é uma **interface**, não uma coleção concreta.
- Estende `ICollection<T>`, adicionando acesso por índice.
- Permite acessar, inserir, alterar e remover elementos utilizando índices.
- A implementação mais comum é a `List<T>`.
- É indicada quando um método precisa trabalhar com qualquer coleção indexada, sem depender de uma implementação específica.
