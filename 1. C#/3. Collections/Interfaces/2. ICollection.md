`ICollection<T>` é uma **interface** que representa uma coleção de elementos que pode ser percorrida e modificada. Ela estende `IEnumerable<T>`, adicionando funcionalidades como adicionar, remover elementos e obter a quantidade de itens da coleção.

Diversas coleções do .NET, como: `List<T>`, `HashSet<T>`, `LinkedList<T>`, `Queue<T>` e `Stack<T>` implementam essa interface.

---
### 2. Qual problema resolve?

Resolve a necessidade de trabalhar com diferentes tipos de coleções que compartilham operações básicas, como adicionar, remover, limpar e contar elementos.

Ao utilizar `ICollection<T>`, um método pode receber qualquer coleção que implemente essa interface, sem depender de uma implementação específica, como `List<T>` ou `HashSet<T>`.

---
### 3. Como funciona?

`ICollection<T>` define um conjunto de operações comuns para coleções modificáveis.

Além de permitir percorrer os elementos por meio do `IEnumerable<T>`, ela disponibiliza membros para adicionar, remover, verificar a existência de elementos e obter a quantidade de itens armazenados.

Por exemplo:

```
ICollection<string> alunos = new List<string>
{
    "João",
    "Maria"
};

alunos.Add("Pedro");

Console.WriteLine(alunos.Count); // 3

foreach (var aluno in alunos)
{
    Console.WriteLine(aluno);
}
```

Neste exemplo:

- A variável foi declarada como `ICollection<string>`.
- A implementação utilizada é uma `List<string>`.
- É possível adicionar novos elementos utilizando `Add()`.
- Também é possível percorrer a coleção utilizando `foreach`.

---
### 4. Quando usar?

- Quando um método precisa adicionar ou remover elementos de uma coleção.
- Quando deseja trabalhar com qualquer coleção modificável.
- Quando não é necessário acessar elementos por índice.
- Quando deseja desacoplar o código de uma implementação específica.

Na prática, `ICollection<T>` é utilizada quando são necessárias operações básicas de manipulação da coleção, sem depender de funcionalidades exclusivas de uma `List<T>` ou outra implementação.

---
### 5. Exemplo prático

```
void AdicionarAluno(ICollection<string> alunos, string nome)
{
    alunos.Add(nome);
}

List<string> lista = new()
{
    "João",
    "Maria"
};

HashSet<string> conjunto = new()
{
    "Pedro",
    "Carlos"
};

AdicionarAluno(lista, "Ana");
AdicionarAluno(conjunto, "Julia");
```

Observe que o mesmo método funciona tanto com uma `List<T>` quanto com um `HashSet<T>`, pois ambos implementam `ICollection<T>`.

---
### 6. Principais membros

|Membro|Descrição|
|---|---|
|`Add()`|Adiciona um elemento à coleção.|
|`Remove()`|Remove um elemento da coleção.|
|`Clear()`|Remove todos os elementos da coleção.|
|`Contains()`|Verifica se um elemento existe na coleção.|
|`CopyTo()`|Copia os elementos para um array.|
|`Count`|Retorna a quantidade de elementos armazenados.|
|`IsReadOnly`|Indica se a coleção é somente leitura.|

---
### 7. Limitações

Apesar de fornecer operações básicas de manipulação, `ICollection<T>` possui algumas limitações.

- Não fornece acesso por índice.
- Não define como os elementos são armazenados.
- Não garante a ordem dos elementos.
- Não oferece métodos específicos de determinadas coleções, como `Push()` (`Stack<T>`), `Enqueue()` (`Queue<T>`) ou `AddFirst()` (`LinkedList<T>`).

---
### 8. Resumo

- `ICollection<T>` é uma **interface**, não uma coleção concreta.
- Estende `IEnumerable<T>`, adicionando operações de modificação da coleção.
- Permite adicionar, remover, limpar e contar elementos.
- É implementada por diversas coleções do .NET, como `List<T>`, `HashSet<T>` e `LinkedList<T>`
- É indicada quando um método precisa manipular uma coleção sem depender de uma implementação específica.
