`IEnumerable<T>` é uma **interface** que representa uma sequência de elementos que pode ser percorrida. Ela **não armazena dados**, mas define um contrato para que uma coleção possa ser iterada utilizando estruturas como o `foreach`.

Grande parte das coleções do .NET, como: `List<T>`, `Array`, `Dictionary<TKey, TValue>`, `HashSet<T>`, `Queue<T>`, `Stack<T>` e `LinkedList<T>`, implementam essa interface.

---
### 2. Qual problema resolve?

Resolve a necessidade de percorrer diferentes tipos de coleções utilizando uma única interface.

Ao invés de escrever métodos específicos para `List<T>`, `Array` ou `HashSet<T>`, é possível trabalhar com `IEnumerable<T>`, tornando o código mais flexível e reutilizável.

Além disso, `IEnumerable<T>` é a base para o funcionamento do **LINQ**.

---
### 3. Como funciona?

`IEnumerable<T>` define que uma coleção pode fornecer um **enumerador** responsável por percorrer seus elementos.

Quando um `foreach` é executado, o compilador utiliza esse enumerador para acessar cada elemento da sequência.

Por exemplo:

```
IEnumerable<string> alunos = new List<string>
{
    "João",
    "Maria",
    "Pedro"
};

foreach (var aluno in alunos)
{
    Console.WriteLine(aluno);
}
```

Neste exemplo:

- A variável foi declarada como `IEnumerable<string>`.
- A implementação utilizada é uma `List<string>`.
- O `foreach` percorre todos os elementos da sequência.
- Não importa qual coleção concreta esteja sendo utilizada, desde que implemente `IEnumerable<T>`.

---
### 4. Quando usar?

- Quando um método apenas precisa percorrer uma sequência de elementos.
- Quando não é necessário adicionar ou remover elementos da coleção.
- Quando deseja escrever métodos que funcionem com diferentes tipos de coleções.
- Quando trabalha com consultas utilizando LINQ.

Na prática, muitos métodos recebem ou retornam um `IEnumerable<T>` para desacoplar o código de uma implementação específica, como `List<T>` ou `Array`.

---
### 5. Exemplo prático

```
IEnumerable<int> numeros = new List<int>
{
    10,
    20,
    30,
    40
};

foreach (var numero in numeros)
{
    Console.WriteLine(numero);
}
```

Saída:

```
10
20
30
40
```

Outro exemplo:

```
void ExibirAlunos(IEnumerable<string> alunos)
{
    foreach (var aluno in alunos)
    {
        Console.WriteLine(aluno);
    }
}

List<string> lista = new() { "João", "Maria" };
string[] array = { "Pedro", "Carlos" };

ExibirAlunos(lista);
ExibirAlunos(array);
```

Observe que o mesmo método funciona tanto com uma `List<T>` quanto com um `Array`, pois ambos implementam `IEnumerable<T>`.

---
## 6. Principais membros

|Membro|Descrição|
|---|---|
|`GetEnumerator()`|Retorna o enumerador responsável por percorrer a sequência.|
|`foreach`|Percorre automaticamente todos os elementos de um `IEnumerable<T>`.|
|Métodos LINQ|Permite utilizar operações como `Where()`, `Select()`, `OrderBy()`, `First()`, `Any()`, entre outras.|

> **Observação:** Diferente das coleções concretas, `IEnumerable<T>` possui apenas um método (`GetEnumerator()`). Os demais métodos frequentemente utilizados, como `Where()` e `Select()`, são **métodos de extensão** fornecidos pela classe `Enumerable` (namespace `System.Linq`).

---
## 7. Limitações

Apesar de ser extremamente útil para abstrair coleções, `IEnumerable<T>` possui algumas limitações.

- Não permite adicionar ou remover elementos.
- Não fornece acesso por índice.
- Não possui informações sobre a quantidade de elementos (como `Count`) sem utilizar métodos adicionais.
- Apenas define como percorrer uma sequência, não como modificá-la.

---
## 8. Resumo

- `IEnumerable<T>` é uma **interface**, não uma coleção concreta.
- Representa qualquer sequência de elementos que pode ser percorrida.
- É implementada pela maioria das coleções do .NET.
- Permite escrever código independente da coleção utilizada.
- É a base para o funcionamento do `foreach` e do **LINQ**.
