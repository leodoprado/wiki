1. Oque é?
	Consiste em **representar apenas as características e comportamentos essenciais de um objeto**, ocultando detalhes desnecessários para quem o utiliza.
	
2. Qual problema resolve?
	Reduz a complexidade do sistema, permitindo que o desenvolvedor se preocupe apenas com o que o objeto faz, e não com todos os detalhes de como ele faz.

3. Como funciona?
	A abstração define uma visão simplificada de uma entidade, expondo apenas o que é relevante para o contexto do sistema.

4. Quando usar?
	Sempre que uma entidade possuir muitos detalhes internos que não precisam ser conhecidos por quem a utiliza.

5. Exemplo prático?
	
	Ao dirigir um carro, você utiliza:
	- Volante
	- Pedais
	- Câmbio
	
	Você não precisa saber exatamente como o motor funciona internamente para dirigir.
	
Na programação é a mesma ideia
```
public class Carro
{
    public void Ligar()
    {
        // Lógica interna complexa
    }
}
```

Utilização:
```
var carro = new Carro();

carro.Ligar();
```
Quem usa a classe apenas chama `Ligar()`. Os detalhes internos ficam escondidos.

A **abstração é um conceito**, enquanto a **classe abstrata é uma ferramenta da linguagem** para ajudar a implementar esse conceito.
```
public abstract class Carro
{
    public abstract void Ligar();
}
```
"Todo carro deve saber ligar."

Agora cada tipo de carro implementa sua própria lógica:
```
public class CarroCombustao : Carro
{
    public override void Ligar()
    {
        Console.WriteLine("Injetando combustível e acionando o motor.");
    }
}
```
```
public class CarroEletrico : Carro
{
    public override void Ligar()
    {
        Console.WriteLine("Ativando bateria e sistema elétrico.");
    }
}
```

Utilização:
```
Carro carro1 = new CarroCombustao();
Carro carro2 = new CarroEletrico();

carro1.Ligar();
carro2.Ligar();
```
