1. Oque é?
	É um contrato que define um conjunto de regras que uma classe deve seguir. 
	Ela diz: “se você implementar essa interface, você é obrigado a ter esses métodos/propriedades”.

Exemplo simples:
```
public interface IPagamento
{
    void Pagar();
}
```

2. Qual problema resolve?
	Antes das interfaces, diferentes classes tinham comportamentos parecidos, mas cada uma implementava de um jeito diferente, o que gerava:
	
	- Código acoplado (dependente de classes específicas)
	- Dificuldade de trocar implementações
	- Dificuldade de testar o sistema
	
	A interface resolve isso criando um **padrão comum de comportamento**.

3. Como funciona?
	Uma classe “assina o contrato” implementando a interface. Ela é obrigada a implementar todos os membros definidos.

Exemplo:
```
public class PagamentoCartao : IPagamento
{
    public void Pagar()
    {
        Console.WriteLine("Pagando com cartão");
    }
}
```
Agora qualquer código que use `IPagamento` não precisa saber qual classe está sendo usada.

4. Quando usar?
	Use interfaces quando:
	- Você quer **desacoplar código**
	- Existem **múltiplas implementações possíveis**
	- Você quer facilitar testes (mock)
	- Vai usar **Injeção de Dependência**
	- Quer trabalhar com **polimorfismo**
	
	Exemplo real:
	- Pagamento com cartão
	- Pagamento com PIX
	- Pagamento com boleto
	
	Todos fazem “Pagar”, mas de formas diferentes.

4. Exemplo prático?

```
public interface IPagamento
{
    void Pagar(decimal valor);
}

public class PagamentoPix : IPagamento
{
    public void Pagar(decimal valor)
    {
        Console.WriteLine($"Pagando {valor} via PIX");
    }
}

public class PagamentoCartao : IPagamento
{
    public void Pagar(decimal valor)
    {
        Console.WriteLine($"Pagando {valor} no cartão");
    }
}
```

Utilização:
```
IPagamento pagamento = new PagamentoPix();
pagamento.Pagar(100);
```
O código não sabe qual implementação está sendo usada.

*Resumo mental:*  
**Interface é um “contrato de comportamento”**
- Não diz COMO fazer
- Só diz O QUE deve ser feito
