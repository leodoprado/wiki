1. Oque é?
	É a capacidade de objetos de diferentes classes responderem à mesma ação de maneiras diferentes. O mesmo método pode ter comportamentos distintos dependendo do objeto que o executa.
	
2. Qual problema resolve?
	Permite criar sistemas mais flexíveis e extensíveis, reduzindo a necessidade de condicionais (`if`, `switch`) para tratar diferentes tipos de objetos.
	
3. Como funciona?
	Uma classe base define um método, e as classes filhas podem fornecer sua própria implementação para esse método.
	
4. Quando usar?
	Quando diferentes entidades possuem o mesmo comportamento conceitual, mas executam esse comportamento de formas diferentes.

	Exemplos:
	- Diferentes formas de calcular impostos.
	- Diferentes tipos de pagamento.
	- Diferentes tipos de veículos se movimentando.
	- Diferentes animais emitindo sons.
	
5. Exemplo prático?

Classe Pai:
```
public class Animal
{
    public virtual void EmitirSom()
    {
        Console.WriteLine("Som genérico");
    }
}
```

Classe Filha:
```
public class Cachorro : Animal
{
    public override void EmitirSom()
    {
        Console.WriteLine("Au Au");
    }
}

public class Gato : Animal
{
    public override void EmitirSom()
    {
        Console.WriteLine("Miau");
    }
}
```

Utilização:
```
Animal cachorro = new Cachorro();
Animal gato = new Gato();

cachorro.EmitirSom();
gato.EmitirSom();
```
