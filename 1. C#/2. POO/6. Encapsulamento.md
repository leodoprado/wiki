1. Oque é?
	Consiste em **proteger os dados de um objeto e controlar como eles podem ser acessados ou modificados**. Em vez de permitir alterações diretas, o próprio objeto define as regras para manipular seus dados.

2. Qual problema resolve?
	Evita que o estado de um objeto seja alterado de forma incorreta, garantindo maior segurança e consistência das informações.

3. Como funciona?
	O acesso aos atributos é restringido por modificadores de acesso (`private`, `protected`, `public`), e as alterações são realizadas através de métodos ou propriedades controladas.

4. Quando usar?
	Sempre que um objeto possuir regras de negócio ou informações que não devem ser alteradas livremente.

5. Exemplo prático?

Sem Encapsulamento:
```
public class ContaBancaria
{
    public decimal Saldo;
}
```

Qualquer pessoal pode alterar o saldo:
```
conta.Saldo = -1000;
```

Com Encapsulamento:
```
public class ContaBancaria
{
    public decimal Saldo { get; private set; }

    public void Depositar(decimal valor)
    {
        if (valor > 0)
            Saldo += valor;
    }
}
```

Agora o saldo não pode ser alterado diretamente:
```
conta.Saldo = -1000; // Erro
```
