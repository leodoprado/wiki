1. Oque é?
	Modificadores de acesso são palavras-chave que definem **quem pode acessar uma classe, método ou propriedade**. Eles controlam o nível de visibilidade do código.

Exemplo:
```
public class Pessoa
{
    private string nome;
}
```

2. Qual problema resolve?
	Sem controle de acesso, qualquer parte do sistema poderia:
	- alterar dados internos de qualquer classe
	- quebrar regras de negócio
	- gerar inconsistência no sistema
	- aumentar acoplamento entre classes

3. Como funciona?
	O C# define níveis de acesso que controlam visibilidade.

	- public -> Acesso liberado para todo o sistema.
	- private -> Acesso apenas dentro da própria classe.
	- protected -> Acesso dentro da classe e das classes filhas.
	- internal -> Acesso apenas dentro do mesmo projeto (assembly).

4. Quando usar?
	Use modificadores de acesso sempre que:
	- quiser proteger dados internos da classe
	- quiser controlar como outras partes do sistema interagem
	- estiver aplicando encapsulamento
	- quiser reduzir acoplamento entre classes

	Regra prática:
	Tudo deve ser private por padrão, e só exposto se necessário.

5. Exemplo prático?

Cenário: Conta bancária
```
public class ContaBancaria
{
    public string Titular { get; private set; }
    public decimal Saldo { get; private set; }

    public ContaBancaria(string titular, decimal saldoInicial)
    {
        Titular = titular;
        Saldo = saldoInicial;
    }

    public void Depositar(decimal valor)
    {
        Saldo += valor;
    }

    public void Sacar(decimal valor)
    {
        if (valor <= Saldo)
            Saldo -= valor;
    }
}
```

Utilização:
```
var conta = new ContaBancaria("Leonardo", 100);

conta.Depositar(50);
// conta.Saldo = 999; ❌ não permitido
```

*Resumo mental:*
**Modificadores de acesso definem “quem pode ver o quê”**.
Eles são a base do encapsulamento na prática.
